# Bitcoin AML Risk Analysis Using Graph Networks

## Overview

This project investigates how graph analytics and network-based risk scoring techniques can support Anti-Money Laundering (AML) investigations within cryptocurrency transaction networks.

Using the Elliptic Bitcoin Dataset, Bitcoin transactions were modeled as a directed graph in which:
- nodes represent transactions
- edges represent transaction flow relationships

The project combines exploratory data analysis, graph network analysis, and rule-based AML scoring techniques to identify unknown transactions with elevated exposure to illicit activity.

---

## Business Problem

Traditional AML transaction monitoring systems often focus on isolated transaction behavior, which may fail to detect suspicious relationships hidden within complex cryptocurrency transaction networks.

This project explores whether graph-based transaction analysis can improve investigative prioritization by identifying unknown transactions connected to illicit activity.

---

## Dataset

Dataset used:
- Elliptic Bitcoin Dataset

Dataset components:
- Transaction features
- Transaction class labels
- Transaction relationship edges

Transaction labels:
- `1` → illicit
- `0` → licit
- `-1` → unknown

The dataset contains more than 200,000 Bitcoin transactions and reflects the highly imbalanced nature of real-world AML investigations.

---

## Methodology

The project was developed using Python in Google Colab and included the following stages:

1. Data preprocessing and cleaning
2. Exploratory Data Analysis (EDA)
3. Directed graph network construction using NetworkX
4. Transaction relationship analysis
5. Degree and betweenness centrality analysis
6. Illicit proximity analysis
7. Rule-based AML risk scoring
8. Suspicious subnetwork investigation

---

## Key Findings

- Illicit transactions represented only a small percentage of total activity, reflecting realistic AML detection challenges.
- Graph analysis revealed unknown transactions directly connected to illicit activity.
- Network centrality metrics identified structurally influential transactions within the transaction network.
- Risk scoring techniques successfully prioritized unknown transactions with elevated suspicious exposure.
- Model refinement reduced false positives generated by highly connected legitimate nodes.
